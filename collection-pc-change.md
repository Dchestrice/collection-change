# 催收台账小程序 - AI 原型生成工程化 PRD

# 一、项目目标

本项目用于生成：

```text
微信小程序风格 HTML 原型
```

系统核心目标：

* 管理催收项目
* 管理收款节点
* 管理延期审批
* 自动催收提醒
* 站内消息通知
* 数据隔离与权限控制

当前阶段：

```text
仅生成前端 HTML 原型
```

不需要：

* 后端接口
* 数据库
* 登录鉴权
* 真正上传能力

系统需使用：

* mock 数据
* 原生 HTML
* TailwindCSS
* 原生 JavaScript

---

# 二、AI生成核心约束（非常重要）

生成代码时必须严格遵守。

---

## 1. 仅允许生成移动端小程序布局

必须：

* 使用移动端宽度
* 微信小程序风格
* 卡片式布局
* 顶部导航栏
* 底部操作栏
* 列表卡片结构

禁止：

* PC后台布局
* 左侧菜单
* 顶部复杂导航
* 大表格布局
* 多栏布局

---

## 2. 页面风格必须统一

统一：

* 白底
* 蓝色主色
* 卡片阴影
* 圆角
* 统一按钮
* 统一状态标签

所有页面：

* 使用同一视觉体系
* 使用同一间距体系
* 使用同一字体层级

禁止：

* 页面风格不一致
* 每页不同UI
* 混合PC风格

---

## 3. 仅允许增量生成

禁止：

* 重构页面
* 修改已有页面结构
* 随意新增容器
* 修改页面路由
* 修改页面命名

后续修改时：

必须：

* 优先复用现有DOM
* 优先复用已有class
* 仅局部增量修改

---

## 4. 页面必须可点击

所有页面必须：

* 可跳转
* 有mock数据
* 有弹窗
* 有Toast
* 有状态切换
* 有表单校验

禁止：

* 静态页面
* 空白页面
* 仅UI截图

# 四、角色权限

# 1. 管理员

权限：

* 查看全部项目
* 编辑全部项目
* 修改负责人
* 审批延期
* 管理提醒规则
* 查看全部审批
* 查看全部消息
* 更新收款状态

权限：

* 查看自己负责项目
* 创建项目
* 编辑自己项目
* 发起延期
* 查看自己审批
* 查看自己消息

规则：

```text
只能查看自己负责项目
```

# 2. 收款节点

一个项目最多：

```text
6个收款节点
```

字段：

| 字段     | 类型   |
| ------ | ---- |
| 应收金额   | 金额   |
| 计划收款时间 | 日期   |
| 收款状态   | 枚举   |
| 逾期状态   | 系统计算 |
| 到账金额   | 金额   |
| 到账时间   | 日期   |
| 差额     | 自动计算 |

---

## 收款状态

```text
待回款
已回款
坏账关闭
```

---

## 逾期状态

```text
未逾期
已逾期
```

规则：

```text
当前日期 > 计划收款时间
且收款状态 != 已回款
则为已逾期
```

---

# 七、状态流转

# 收款状态流转

```text
待回款
  ↓
已回款
  ↓
坏账关闭
```

规则：

## 修改为已回款时必须填写：

* 到账金额
* 到账时间

---

## 坏账关闭后

管理员允许重新修改状态。
